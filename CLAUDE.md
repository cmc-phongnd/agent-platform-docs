# CLAUDE.md — CMC Agent Platform (CAP) Product Docs

Hướng dẫn ngữ cảnh cho Claude / agent khác khi làm việc trong repo này.

## Repo là gì

Đây là **product documentation site** cho **CMC Agent Platform (CAP)** — nền tảng multi-tenant cho việc xây dựng AI agent & workflow trong doanh nghiệp. Repo **không chứa code sản phẩm**, chỉ chứa docs (Markdown/MDX) được build bằng [Docusaurus 3.10](https://docusaurus.io/).

- **Title**: CMC Agent Platform (CAP)
- **URL production**: `https://cap.cmc.local` (placeholder — đổi khi deploy)
- **Locale mặc định**: `vi` (tiếng Việt) — toàn bộ nội dung viết bằng tiếng Việt; thuật ngữ kỹ thuật giữ tiếng Anh (Agent, Workflow, Tenant…)
- **Diagram**: Mermaid (đã enable trong [docusaurus.config.ts](docusaurus.config.ts))
- **Sidebar**: autogenerated từ `docs/` — file prefix số quyết định thứ tự (xem [sidebars.ts](sidebars.ts))

## Quy ước viết docs

- **Ngôn ngữ**: tiếng Việt cho nội dung; thuật ngữ kỹ thuật giữ tiếng Anh nguyên gốc.
- **Frontmatter**: mỗi file md bắt đầu bằng `--- sidebar_position: N ---`.
- **Tên file**: prefix số 2 chữ số (`01-`, `02-`…) trùng với `sidebar_position` — vì [docusaurus.config.ts:42](docusaurus.config.ts#L42) bật `numberPrefixParser: false`, prefix là **một phần slug**.
- **Category**: mỗi thư mục con có `_category_.json` với `label` (ví dụ `"01 — Overview"`), `position`, và `link` (thường là `generated-index`).
- **Trạng thái nội dung** (đặt ngay sau heading):
  - 🟢 **Đã thống nhất** — sẵn sàng triển khai
  - 🟡 **Dự thảo** — đang hoàn thiện, có thể điều chỉnh
  - 🔴 **Khung sườn** / **Placeholder** — mới có tiêu đề và câu hỏi
- **Internal link**: dùng absolute path không có `.md`, ví dụ `[IAM & RBAC](/02-domain/02-iam-rbac)`. `onBrokenLinks: 'warn'`.
- **Mermaid**: viết trong code fence ` ```mermaid `. Tránh ký tự đặc biệt chưa escape (`&`, `<`, `>`) trong label node — dùng `&amp;` / quote string.
- **Bảng**: ưu tiên Markdown table cho mọi thông tin dạng so sánh / mapping.

## Lệnh thường dùng

```bash
pnpm install            # Cài deps (repo dùng pnpm — có pnpm-lock.yaml)
pnpm start              # Dev server localhost:3000, hot reload
pnpm build              # Build static → ./build
pnpm typecheck          # Kiểm TypeScript trong src/
pnpm clear              # Xoá .docusaurus cache khi build lỗi
```

> **Lưu ý**: README.md ghi `yarn` nhưng project thực tế dùng **pnpm** (xem [pnpm-lock.yaml](pnpm-lock.yaml) và commit `Sync pnpm-lock.yaml`). Khi sửa deps phải chạy `pnpm install` và commit lock file.

## Truy cập docs từ agent ngoài (MCP)

Nếu agent khác (Claude Code session khác, agent SDK, IDE plugin…) cần đọc docs này:

- **3 cách kết nối** (hosted service / `fetch` / `filesystem` local — theo độ khuyến nghị): xem [docs/09-agent-access/01-mcp-access.md](docs/09-agent-access/01-mcp-access.md).
- **Custom MCP server `cap-docs-mcp` (hosted)**: spec thiết kế ở [docs/09-agent-access/02-custom-mcp-spec.md](docs/09-agent-access/02-custom-mcp-spec.md) — tool list, schema, deploy config. Chưa implement.

> ⚠️ Path filesystem trên máy cá nhân **không** phải config dùng chung — đừng commit/share path cụ thể. Xem nguyên tắc thiết kế ở [01-mcp-access#nguyên-tắc-thiết-kế](docs/09-agent-access/01-mcp-access.md).

## Khi sửa docs — checklist nhanh

1. File mới: tạo trong đúng section, frontmatter `sidebar_position` khớp prefix số.
2. Thêm mục vào trang `intro.md` "Cấu trúc tài liệu" nếu là section level top.
3. Nếu thêm thuật ngữ mới → cập nhật [docs/01-overview/03-glossary.md](docs/01-overview/03-glossary.md) (skill `glossary-sync` giúp việc này).
4. `pnpm start` để verify render + Mermaid không lỗi parse, không có broken link.
5. Không commit `build/`, `node_modules/`, `.docusaurus/` (đã ignore).
